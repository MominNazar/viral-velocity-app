Project: Viral Velocity Engine
Version: 1.0
Date: TBD
Prepared by: Manaknight Inc.
1. Executive Summary
The Viral Velocity Engine is a mobile application designed to help users decide which photos to post on social media by providing a social score for each picture. The app also enables AI-powered photo enhancement to improve image quality and boost scores. Users can upload multiple photos, view AI enhancements, and make data-driven posting decisions. An admin web portal will provide oversight of users, subscriptions, and metrics.

3. Business Requirements
3.1 Functional Requirements
Requirement ID
Description
FR-1
Users can sign up, log in, and reset passwords (OTP flow for reset; Remember Me on login).

FR-3
Sign-up triggers welcome email.
FR-4
If DOB < 18, show a parental supervision popup that must be accepted to proceed.
FR-5
Guest login is disabled; all users must register to access the app.
FR-6
Free trial: 5 photos during the first 30 days after sign-up (one-time trial only).
FR-7
Users can upload 1–5 photos per batch from gallery; selection limit enforced with clear message.
FR-8
Users can capture a new photo via camera and request immediate scoring.
FR-9
Each photo shows a main Social Score (0–100).
FR-10
Score Details collapsible shows sub-scores: Technical Quality, Composition, Emotional Resonance, Storytelling, Color Psychology.
FR-11

Enhance generates up to 5 enhanced versions, each with its own Social Score.

Users can swipe right (save) or left (discard) enhanced versions; saved ones persist to Library.
FR-14

FR-15

The app does not alter personal appearance (e.g., make someone thinner/celebrity-like).
FR-17
Content moderation blocks violent/vulgar/inappropriate images; tolerance is configurable.
FR-18
Library lists all originals and enhanced outputs; user can filter by Score or Date with ASC/DESC ordering.
FR-19

Subscription page shows Monthly and Annual plans and highlights current plan with expiry date.

FR-27
Logout and (if present) Disable Profile operate per wireframe; login credentials are retained after cancellation.

FR-32
Data Self-Deletion (Admin) allows setting Month/Day retention window; Save/Cancel updates policy.

Subscribers Page (Admin) lists Username, Email, Plan Type, Renewal Date, Status; Search, Export, and Active/Inactive filters.
FR-34
Admin can Activate/Deactivate users with confirmation prompts.

FR-36

3.2 Non-Functional Requirements
Requirement ID
Description
NFR-1
Performance: Scoring returns results within an acceptable time under normal load.
NFR-2
Performance: Enhancement returns results within an acceptable time for up to 5 images concurrently.
NFR-3
Robustness: Attempts to upload >5 images are blocked gracefully with clear error; no crashes/hangs.
NFR-4
Scalability: Concurrent users can upload/score without materially impacting one another.
NFR-5
Reliability: Admin Search and Export complete without timeout for dataset sizes implied by wireframe tables.
NFR-6
Security: Password-reset OTP is rate-limited; invalid attempts are throttled.
NFR-7
Security: 2FA (Admin) enforces a second factor on the next login after enabling.
NFR-8
Privacy: Data Self-Deletion job purges images after configured retention; actions are logged.
NFR-9
Privacy: User-initiated image deletion (Library) and admin deletion requests are honored per retention settings.
NFR-10
Moderation: Tolerance threshold is configurable; blocked images are not stored, scored, or enhanced.
NFR-11
UI Consistency: Score badge appears top-right on images where specified; consistent across screens.
NFR-12
UI Consistency: Score Details collapsible behaves the same for original and enhanced images.
NFR-13
Usability: Side-by-Side Compare remains legible and responsive on common mobile sizes.
NFR-14
Usability: Swipe interactions on enhancement stack are responsive with low gesture error rate.
NFR-15
Data Integrity: Saved/Discarded states persist across app restarts and appear in Admin as Passed/Failed.
NFR-16
Correctness: Library filters and ordering (Score/Date; ASC/DESC) return stable, correct results.
NFR-17
Access Control: Non-subscribers/expired trials are blocked from app functionality and prompted to subscribe.
NFR-18
Subscription State: Cancel flow shows exact term end; after term, access is blocked unless re-subscribed; data deletion occurs on schedule unless re-subscribed before deadline.
NFR-19
Error Handling: Clear, user-friendly messages for moderation blocks, selection-limit errors, and paywall blocks.
NFR-20
Compatibility: Mobile app supports the target platforms (iOS/Android) and expected gallery/camera permissions behavior.

Forgot Password (OTP Flow)
User flow

Back from OTP returns to Enter Email; Back from Enter Email returns to Login.

Flow Diagram:
Admin → Images Matched
User flow
From Admin Dashboard, the admin opens Images Matched.
Search bar allows filtering by User, Plan, and Avg scored; clicking Search refreshes the Overview table.
Each row in Overview shows: User, Plan, Images Scored, Avg. Image Score, Last Active, and Action icons:
View → opens the Detail View for that user/session with the uploaded image and all enhanced images marked Passed/Failed with their scores (passed = user accepted; failed = user discarded).
Delete → confirmation to delete that user’s images/data (per earlier requirement for admin-side deletion).
Data Self-deletion setting → opens a modal to configure the retention window (Months/Days). Save updates policy; Cancel closes without changes.
Export → downloads the current dataset (respecting filters) for offline analysis.
